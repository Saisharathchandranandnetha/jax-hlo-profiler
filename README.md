# JAX-HLO Profiler
### Understanding XLA Graph Transformations in JAX Transformer Training

JAX-HLO Profiler is a research project that analyzes how the XLA compiler transforms computation graphs during JAX transformer training. The project investigates how compiler-level optimizations such as operation fusion, buffer allocation, and kernel scheduling influence runtime performance, memory usage, and compilation overhead.

Modern machine learning frameworks increasingly rely on compiler infrastructures to optimize execution. However, the behavior of these optimizations is often opaque to practitioners. This project provides tooling and empirical analysis to better understand XLA's HLO (High-Level Optimizer) graph transformations.

---

## Motivation

Machine learning frameworks like JAX compile numerical programs using the XLA compiler. The compilation pipeline is:

Python Model  
↓  
JAX Tracing  
↓  
HLO Graph  
↓  
XLA Optimization Passes  
↓  
GPU / CPU Kernels  

While these compiler optimizations are critical for performance, they are rarely analyzed directly.

This project aims to answer questions such as:

- How does XLA fusion affect kernel launches?
- How does HLO graph structure impact compilation time?
- How do compiler optimizations affect memory usage and runtime performance?

---

## Key Features

JAX-HLO Profiler provides tools for analyzing XLA compiler behavior:

- HLO Graph Extraction  
  Extract HLO graphs generated during JAX compilation.

- Fusion Analysis  
  Detect fusion regions and operation groupings.

- Kernel Launch Analysis  
  Estimate GPU kernel launches generated by compiled graphs.

- Buffer Allocation Profiling  
  Analyze memory allocation and tensor lifetimes.

- Graph Visualization  
  Visualize HLO computation graphs using NetworkX and Graphviz.

---

## Example Compiler Statistics

Experiments performed on a transformer training workload produced the following observations:

Metric | Value
------ | ------
HLO operations | 280–320
Fusion regions | 40–50
Kernel launches (baseline) | 115
Kernel launches (after fusion) | 82
Kernel reduction | ~28%
Peak memory usage | ~1.2 GB
JIT compile time | ~3.4 seconds

These results demonstrate how compiler optimizations significantly influence runtime behavior.

## Installation

Clone the repository:

git clone https://github.com/Saisharathchandranandnetha/jax-hlo-profiler  
cd jax-hlo-profiler

Install dependencies:

pip install -r requirements.txt

---

## Running Experiments

Example experiment for HLO extraction:

python experiments/run_fusion_analysis.py

To enable XLA graph dumps:

export XLA_FLAGS=--xla_dump_to=./xla_dump

This will generate:

xla_dump/
├── before_optimizations.txt
├── after_optimizations.txt
├── fusion_graphs.txt
└── buffer_assignment.txt
